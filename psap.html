What is an NG911 PSAP?

A Next Generation 911-enabled Public Safety Answering Point (PSAP) represents the evolution of the traditional 911 call center. Instead of relying solely on analog phone lines and basic location displays, NG911 PSAPs utilize IP-based workstations and infrastructure connected to the ESInet.

This allows them to receive not only voice calls but also text messages (RTT, SMS), images, video feeds, and data from various sources (like IoT devices or telematics systems). They integrate tightly with NGCS for call routing and features, and often feature advanced mapping displays and Computer-Aided Dispatch (CAD) systems capable of handling multimedia information.

Role of PSAP in Emergency Call Handling

Inputs and Operation

NG911 PSAPs handle incoming emergency communications and coordinate responses:

  • Data Inputs: Receives incoming sessions (voice calls, RTT, SMS, video) via SIP and associated media streams (e.g., RTP) from the ESInet/NGCS. Receives validated location data (coordinates, civic address) and potentially supplementary data (caller name, device type, medical info, telematics data) associated with the session.
  • Control Signals: Receives SIP messages for session control (setup, teardown, hold, transfer, conference). Interacts with the local CAD system, typically via APIs, to log incidents, query resources, and record dispatch actions. Sends SIP requests to NGCS to initiate actions like conferencing external parties or transferring calls.
  • Operation:
    1. Call Taker workstation receives notification of an incoming session via SIP from the ESInet/NGCS.
    2. Call Taker answers the session (voice, text, etc.).
    3. Associated location and supplementary data are displayed, often plotted on an integrated GIS map interface.
    4. Call Taker communicates with the caller, gathering details.
    5. Incident information is entered into the CAD system.
    6. Call Taker/Dispatcher uses CAD to identify and dispatch appropriate responders (police, fire, EMS).
    7. May use NGCS features (via SIP requests) to conference in translators, specialists, or transfer the call if needed.
    8. Session is logged and eventually terminated.
  • Outputs: Generates dispatch instructions and resource assignments via the CAD system, which are communicated to field responders. Sends SIP requests to NGCS to initiate conferences, transfers, or call terminations. Provides feedback and updates to the caller during the session. Logs call details, media, and actions taken for record-keeping and analysis.

Examples of NG911 PSAP Functionality

  • Handling an RTT Call: The PSAP workstation receives an incoming Real-Time Text session. The call taker engages in a text conversation with the caller, while their location is automatically displayed on the map screen, allowing for efficient dispatch even without voice communication.
  • Utilizing Bystander Video: During a multi-vehicle accident call, a bystander offers to send video. The PSAP provides a secure link (potentially via NGCS), receives the video stream, and displays it on the call taker/dispatcher screen, providing valuable real-time situational awareness.
  • Processing Telematics Data: Following a vehicle crash, the PSAP receives an automated notification from the car's telematics system via the ESInet. This data includes precise location, airbag deployment status, and impact severity, which appears directly in the CAD incident alongside the subsequent voice call from the driver, allowing for a more informed dispatch.

Key Features of NG911 PSAPs

  • Multimedia Support: Voice, text, images, video, sensor data.
  • Advanced CAD Integration: Enhanced dispatch and resource management.
  • Conferencing: Seamless collaboration with external experts and agencies.
  • Interoperability: Easy call transfers and data sharing across jurisdictions.
  • Resilience: Redundant systems and failover capabilities.

Conclusion

NG911-enabled PSAPs are the front line of emergency response, leveraging modern technology to improve communication, coordination, and outcomes.
