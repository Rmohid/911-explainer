What is ESInet?

The Emergency Services IP Network (ESInet) is a private, secure, managed IP-based network specifically designed for emergency communications. It serves as the foundational transport layer for NG911, replacing the legacy network of analog circuits and selective routers. Unlike the public internet, the ESInet is a controlled environment with guaranteed quality of service, high availability, and robust security measures.

It interconnects various entities involved in emergency response, including Public Safety Answering Points (PSAPs), Next Generation Core Services (NGCS) functional elements, originating networks (via Border Control Functions), and potentially other authorized agencies or services.

Role of ESInet in Emergency Call Handling

Inputs and Operation

The ESInet operates based on standard Internet Protocol (IP) principles but within a controlled environment. Its primary inputs are IP packets containing emergency communications data and control signals:

  • Data Inputs: Encapsulated emergency communications, including VoIP packets (voice), RTT messages (text), video streams, images, sensor data (e.g., from IoT devices or telematics), and CAD data. This data typically enters the ESInet via a Border Control Function (BCF) from originating networks.
  • Control Signals: Protocols like Session Initiation Protocol (SIP) are used to establish, manage, and terminate communication sessions (calls, text sessions) across the network. Routing protocols (e.g., BGP, OSPF) manage the paths data takes across the ESInet.
  • Operation: The ESInet uses IP routing to forward packets towards their destination (e.g., an NGCS function or a specific PSAP). Quality of Service (QoS) mechanisms prioritize emergency traffic over less critical data. Redundant paths and equipment ensure high availability and resilience against failures.
  • Outputs: Delivers IP packets (containing voice, text, data, video, control signals) to the next hop or final destination within the ESInet, such as an NGCS functional element, a PSAP gateway, or another interconnected ESInet via a BCF.

Examples of ESInet Functionality

  • VoIP Call with Location: When a user makes an NG911 call from a smartphone app, the VoIP packets and precise location data (e.g., GPS coordinates) are encapsulated into IP packets. These packets traverse the ESInet, guided by NGCS routing decisions, to reach the appropriate PSAP.
  • Real-Time Text (RTT) Session: An RTT message initiated by a caller enters the ESInet. The network transports the text packets reliably, maintaining the session context via SIP signaling, allowing the PSAP call taker to have a real-time text conversation.
  • Inter-PSAP Data Transfer: During a major incident requiring mutual aid, CAD incident data needs to be shared between two PSAPs. The ESInet provides the secure IP pathway for this data transfer, ensuring both PSAPs have synchronized information.

Key Features of ESInet

  • Supports Multimedia: Voice, text, images, video, sensor data.
  • Secure and Managed: Encrypted, monitored, resilient.
  • Interoperable: Connects multiple PSAPs, agencies, and services.
  • Redundant: Multiple paths to avoid single points of failure.
  • Scalable: Can grow with demand and new technologies.

Conclusion

ESInet is the foundational network infrastructure enabling the advanced capabilities of NG911, ensuring reliable, secure, and flexible emergency communications.
