What is DMCC?

Device, Media, and Call Control (DMCC) is an Avaya API and protocol suite that allows applications to control telephony devices, manage media streams, and handle call signaling. It is part of the Avaya Aura Application Enablement Services (AES) platform.

In NG911 PSAPs, DMCC is often used to integrate call handling software with physical or virtual phone devices, enabling advanced features like call control, media recording, and device status monitoring.

Inputs and Operation

  • Data Inputs: DMCC receives commands from PSAP applications to perform actions such as make/answer/release calls, hold/unhold, conference, transfer, and to start/stop media streams.
  • Control Signals: Uses CTI (Computer Telephony Integration) protocols over AES to control devices, manage call states, and handle media streams (RTP).
  • Operation:
    1. PSAP application connects to AES server via DMCC API.
    2. Application requests control of a device (physical phone, softphone, virtual endpoint).
    3. Application can then monitor device status, control calls (answer, hold, transfer), and access media streams for recording or analysis.
    4. Media streams can be forked to recording servers or analytics tools.
  • Outputs: Device status updates, call state events, media streams (RTP), and control acknowledgments sent back to the PSAP application.

Examples of DMCC Usage

  • Call Control: Automatically answer an incoming 911 call on a call taker's softphone, place it on hold, or transfer it to another operator.
  • Media Recording: Fork the audio stream of a 911 call to a recording server for compliance and review.
  • Device Monitoring: Detect when a call taker's phone goes off-hook, is idle, or encounters an error, enabling better resource management.
